# 🛡️ Shopify Development Standards & Rules of Engagement

This document serves as the **Source of Truth** for all frontend customizations on the Shopify theme within the Antigravity environment. These rules ensure that the store remains functional, scalable, and safe from core logic breakage.

---

## 1. Core Principles
- **Separation of Concerns**: Custom code must be modular. Never "hard-code" logic directly into core Shopify files if a snippet or section can be used instead.
- **Prefixing**: All custom files created by Antigravity must start with the prefix `ag-` (e.g., `sections/ag-hero.liquid`, `assets/ag-custom.css`).
- **Data Integrity**: We do not manually modify `settings_data.json` or any database-critical files in the `/config` folder.

---

## 2. Low-Touch Zones (The "No-Fly List")
These files/folders should only be modified with extreme caution and never deleted.

| Area | Protected Files | Reasoning |
| :--- | :--- | :--- |
| **Layout** | `layout/theme.liquid` | Contains all app injections and core headers. Only append/prepend. |
| **Config** | `/config/*.json` | Stores theme editor settings. Manual changes can break the UI. |
| **App Logic** | `sections/vitals-*.liquid`, `sections/pagefly-*.liquid` | Generated by external apps. Manual edits will be overwritten. |
| **Checkout** | Internal Shopify Scripts | Shopify handles payments/checkout securely. We stay on the frontend. |

---

## 3. Mandatory Tags & Hooks
To ensure Shopify apps and analytics continue to work, the following Liquid tags must **never** be removed:
- `{{ content_for_header }}` (Must remain inside `<head>`)
- `{{ content_for_layout }}` (Must remain inside the main `<main>` wrapper)
- `<form action="/cart/add">` (Standard form action for commerce logic)

---

## 4. Technical Workflow
- **CSS**: All custom styles should be placed in `assets/ag-custom.css`. This file is linked once in `theme.liquid`.
- **Sections**: New layout features should be built as independent **Sections** to allow the user to move them around in the Shopify Theme Editor.
- **JS**: Custom logic should be modularized in `assets/ag-scripts.js` or within specific Section files.

---

## 5. Navigation & Troubleshooting
- **To change a page layout**: Use the `/templates/*.json` file to identify which sections are currently active.
- **To find specific logic**: Search for `{% render 'filename' %}` to trace how snippets are included.
- **To safely test**: Ensure the Shopify CLI is running (`shopify theme dev`) to preview changes in a local environment before pushing to GitHub.
